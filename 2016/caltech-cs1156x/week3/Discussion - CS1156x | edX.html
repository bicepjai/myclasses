          This is how I found the solution to Problem 9. It is closely related to Problems 6 and 8.

          The positive and negative rays mentioned in Lecture 6 Q&A between 1:16:00 and 1:18:10 prompted me to look in 2 directions: positive and negative intervals on the one hand, and, on the other hand, after realising that +-rays are actually the one dimensional perceptron, I started wondering whether there was any relationship between rays and n-intervals, and the 2 dimensional perceptron.

          They turned out to share many properties (but to my initial surprise, less with +- rays):

          • There is a strong relation between the number of regions (intervals) and the break point.

          • One can think of a one dimensional dichotomy as consisting of beads of two colours through a thread, and the two dimensional dichotomies are the resulting necklaces. In 2D the beads at the ends will end up either in the same region or in neighbouring regions.

          • For a given N, the two dichotomies where all neighbouring points have alternating outputs seem to be specially difficult to classify correctly, as they demand a dedicated region for each point, so I focused on one of these dichotomies only, considering it critical.

          • If such a dichotomy can be classified correctly, extending it with one point anywhere and with any output will never break the hypothesis set, because the new point will always be absorbed into a previously existing region.

          • The separators in +- intervals and the lines intersections with the circle are the same thing.

          • A few more, but given that you have kindly read this far I don't want to bore you (even more ;).

          The point of all of this is that Problem 9 happens to be the 3-perceptron, because at the end of the day convex sets are made of the very lines that perceptrons use to separate regions.

          Of course it can be generalized: an s sided convex set as an instance of an s-perceptron. It helps if the points are all evenly separated on a circle.

          How the M perceptrons combine to classify the points is irrelevant at this stage, now all that matters is the hypothesis set. They are just lines that separate the plane into different regions.

          I am not 100% sure of any of this and are lacking the mathematical muscle to proof anything, but I tried to generalise this analysis in 2D anyway. I wonder what would happen in higher dimensions. I guess in 3D the circle becomes a sphere, but the connection is not so clear to me anymore.

              Well! That's pretty complete! The only thing I can add to these posts is a slightly different way of dealing with the growth function for the two-intervals hypothesis set (Question 7).

              We have four boundaries to assign to segments and we can view this situation as involving just three mutually exclusive possibilities. If we assign all the boundaries to one segment, we have no positive interval. If we assign two or three boundaries to the same segment (or interchange the intervals to which the internal boundaries belong), two of the four boundaries become redundant and we have exactly one positive interval. Otherwise, the boundaries are in four distinct segments and we have exactly two positive intervals. The number of hypotheses that can be generated from these three possibilities is:

              • No distinct interval: choose no segments from N+1 segments, (N+10)=1;
              • One distinct interval: choose two distinct segments from N+1 segments , (N+12);
              • Two distinct intervals: choose four distinct segments from N+1 segments, (N+14)

              As these three possibilities are disjoint, we can sum the terms. Viewed from the perspective of non-overlapping events, the argument is easily generalized to M intervals.

                Problem 7

                This one is easy if you recognize that it has optimal substructure, and you remember Yaser's advice: you just need to make sure you count everything and don't count anything twice

                In the spirit of a Dynamic programming problem, let's start at the core and build up.

                0.) consider positive interval problem with 0 (blue) interval ends allowed. Result: everything is red. (N+10)

                1.) consider positive intervals problem with 1 (blue) interval ends allowed. You need 2 bookends to have a legal interval, so in effect we still have zero legal intervals. Thus (N+10). We do not want to count the zero case twice hence we know we are done

                2.) consider positive intervals problem with 2 (blue) interval ends allowed. As discussed in lecture videos, this can be thought of as sampling 2 items without replacement from N+1 slots, (N+12). Now in that special case of both items ending up in the same spot, you in effect get the 1 interval end problem, so we need to add its value in. Again, remember the 1 interval end case already covers the 0 interval end case, and we don't want to double count!

                Thus (N+12)+(N+10).

                3.) consider positive intervals problem with 3 (blue) interval ends allowed. If you draw a picture, you'll realize that this looks identical to the 2 blue interval ends case. Thus (N+12)+(N+10)

                4.) consider positive intervals problem with 4 (blue) interval ends allowed. We can again think of this as having N + 1 spaces to choose from, and wanting to select 4 from there, without replacement. Thus (N+14). Now, what if there is a collision? Recognize the simplest case is when two blue interval ends collide, and you in effect reduce your problem to 3 blue interval ends -- so we add (N+12)+(N+10). Now we may be tempted to add more to this sum, but remember we don't want to double count. The 3 interval ends problem already includes all possible smaller interval ends problems. (Put differently: the 2 interval ends problem is already incorporated in the 3 interval ends problem. And the 1 interval ends problem is in the 2 interval ends, and the 0 interval ends is in the 1 interval ends. Each bigger problem for size N >0 builds on the problem for size N-1.)

                We have now counted all possible outcomes once, thus: (N+14)+(N+12)+(N+10)
