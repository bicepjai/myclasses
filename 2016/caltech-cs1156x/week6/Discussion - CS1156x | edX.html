          Here's my solution to problems 9 and 10, formulating this as a quadratic program, using Julia.

          http://pastebin.com/jVrgj13i

          To my understanding, all variables (xi in the program) are restricted to [0,1], and the objective function is convex so long all xi≥0 -- i.e. this should guarantee polynomial time solutions for any of the problems thrown at it.

          This returns an integral solution for any given solution that has (n_states + initial_nodes)%2 ==0 -- e.g. in our problem we had an even sum of 36 +10 = 46.

          (I have some intuition for why this occurs though I've found anything approaching a proof elusive. Given that we in effect have Markov property making values in one layer only depend on values one layer ahead and one layer behind, I feel like there is an exchange argument lurking somewhere.)

          edit: I should add that I have a lot of xi in there, and they are all restricted to [0,1] because I -- in effect -- modeled the indicator variables associated with each layer. In its pure form indicator variables are binary and only take on {0,1}. It may be more computationally efficient or easy to read to not use indicator variables on this task -- though in general they let you do some very interesting things, so I'm in the habit of using them.

          3. Here is the Python code I wrote yesterday to actually implement the recursive method that ftudone suggested to solve questions 9 and 10. I actually used trial and error and that got me the answers surprisingly quickly, using the logic that 18 x 18 is probably the largest number. And then trying some configurations around that. Actually the size of the input layer 10 makes the amount of weights bigger by combining it with a larger first layer.

            The output of the code is the following:

            The config with minimum weigths is: 10-2-2-2-2-2-2-2-2-2-2-2-2-2-2-2-2-2-2-1   => 46
The config with minimum weigths is: 10-22-14-1   => 510

            The Python code:

            import sys

def splitup(list, prefix, N):
    #add N without further splitup.
    if N >= 2:
        list.append(prefix + '-' + str(N))
    #add N with splitup. Note: last element N-1 in range not included in Python.
    for i in range(2, N-1):
        splitup(list, prefix + '-' + str(i), N - i)

def getWeights(config):
    items = config.split('-')
    total = 0
    for i in range(0, len(items)-1):
        if items[i+1] == '1':
            total += int(items[i])
        else:
            total += (int(items[i]) * (int(items[i+1]) - 1))
    return total


N = 36
prefix = '10'
postfix = '-1'

min = sys.maxsize
max = 1
minConfig = ''
maxConfig = ''
list = []
splitup(list, prefix, N)
for i in range(0, len(list)):
    list[i] += postfix

for config in list:
    w = getWeights(config)
    if w < min:
        min = w
        minConfig = config
    if w > max:
        max = w
        maxConfig = config

print 'The config with minimum weigths is: ' + minConfig + '   => ' + str(min)
print 'The config with minimum weigths is: ' + maxConfig + '   => ' + str(max)

          5. Hi Derek. As a more general example,

            max∑i,j,k,lfikdjlxijxkl
            subject to: xij∈{0,1}& some conditions
            we can convert this QP to LP by introducing new variables:
            zijkl≡xijxkl∈{0,1}

            Since

            zijkl=1⟺(xij=1)&(xkl=1)
            we need additional constraints
            xij≥zijkl and xkl≥zijkl for ⟹part
            xij+xkl≤zijkl+1 for ⟸ part
            .

            The original problem consists of linear and quadratic parts and quadratic part could be converted to linear in this way.

              HW6 PROBLEM 1

              When we utilize a subset H′ of H we reduce the capacity of the model to catch the target function f because some models approximate f better than others. This increases deterministic noise because in general the best hypothesis that approximate f in H′ gets an error (the gray area in figure of the slide 11-16) greater than the one in H, i.e., we can think at the problem of approximating y=sinπx using y=ax+b (less deterministic error) or y=b (greater deterministic error).

              HW6 PROBLEM 7

              It's better to write down the polynomial forms we encounter in the problem (recall L0(x)=1).
              For the general hypothesis:

              H1=w0+w1L1(x)H2=w0+w1L1(x)+w2L2(x)H3=w0+w1L1(x)+w2L2(x)+w3L3(x)H4=w0+w1L1(x)+w2L2(x)+w3L3(x)+w4L4(x)

              For the constrained hypothesis:

              H(10,0,3)=w0+w1L1(x)+w2L2(x)H(10,0,4)=w0+w1L1(x)+w2L2(x)+w3L3(x)H(10,1,3)=w0+w1L1(x)+w2L2(x)+10∑q=3Lq(x)H(10,1,4)=w0+w1L1(x)+w2L2(x)+w3L3(x)+10∑q=4Lq(x)

              Now it's easy to see that for H(10,0,4) when w3=0 we get all the hypothesis of H(10,0,3) so H(10,0,3)∩H(10,0,4)=H2.

              HW6 PROBLEM 8 (Incorrect)

              We have products of the form w(l)ijx(l−1) during the forward process of the output calculation when we compute inputs for the next layer. One input needs the summation of a number of terms equal to the number of neurons in the previous layer. For layer 1 we need 3 inputs and for layer 2 only 1, so we have a total of 5⋅3+3=18 operations.

              During the backpropagation we must compute δs where we find operations such as w(l)ijδ(l)j (always in summation like the previous process). We need 1 δ for the output unit, but this is operation free, 3 for layer 1 and 5 for layer 0, so this gives 3+5⋅3=18 operations.

              Finally we update weights and we need to compute one product of the form x(l−1)iδ(l)j for each weight (we must include this time even the weight for x0=1) and this gives 4+6=10 operations.

              In total we compute 46 operations.

              HW6 PROBLEM 9-10

              Here I used a bit of intuition supported by some tests. The structure of the network is not symmetric because we have 10 inputs and only 1 output, so when the layer 1 has lots of units we should get lots of connections just from the inputs layer and the layer 1. At the same time when we "linearize" the structure (lots of layers with few neurons) we should reduce the number of connections.
              From these assumptions for problem 9 I tried the longest possible structure: 10 inputs, 18 hidden layers with 2 units each (one unit is always x=1) and 1 output, and this gives 10+2⋅18=46. If we reduce the number of hidden layers, i.e. to 17, with 2 units in the first 16 layers and 4 in the last one, we get 52 connections: the number increases, so maybe the intuition it's true :D
              So for problem 10 as first attemp I tried 36 units in 1 hidden layer, but here we lose the benefit of a second layer complexity, and this gives 10⋅35+36=386 connections. Let's try 2 layers, this should be the best we can do according to the assumptions. In this case, if we say x1+1 is the number of units in layer 1 and x2+1 is the number of units in layer 2 (because 1 unit is always fixed), in general we have:

              10x1+(x1+1)x2+(x2+1)=x1x2+10x1+2x2+1

              this is a non-linear function (a quadratic one) and we need some specific tools for integers optimization to maximize (or minimize the negative function as the solution does from now). In any case we can try to minimize using a tool for real solutions, for example gradient descent, so we have a starting point where to search integer solutions. I used fmincon in Matlab so I can express constraints for the variables and the result is x1=21 and x2=13, so respectively 22 and 14 as total number of units, for a total of 510 connections. If you increase the number of layers and apply the minimization you continue to get less connections.

              6. HW6 PROBLEM 8 (Fixed)

                The neural network has 6 inputs in layer 0, 4 neurons in layer 1 and 1 output.

                We have products of the form \(w_{ij}^{(l)}x^{(l-1)}\) during the forward process of the output calculation when we compute inputs for the next layer. One input needs the summation of a number of terms equal to the number of neurons in the previous layer. For layer 1 we need 3 inputs and for layer 2 only 1, so we have a total of \(6 \cdot 3+4 \cdot 1=22\) operations.

                During the backpropagation we must compute \(\delta\)s where we find operations such as \(w_{ij}^{(l)}\delta_{j}^{(l)}\) (always in summation like the previous process). We need 1 \(\delta\) for the output unit, but this is operation free, and 3 for layer 1, so this gives \(3\) operations.

                Finally we update weights and we need to compute one product of the form \(x_{i}^{(l-1)}\delta_{j}^{(l)}\) for each weight. The number of weights is \(6 \cdot 3+4 \cdot 1=22\) and this gives \(22\) operations.

                In total we compute \(22+3+22=47\) operations.
