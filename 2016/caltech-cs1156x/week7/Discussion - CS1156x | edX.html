            HW7 Problem 6
            In the scripts file you can find a simulation to see the correct result, here I give an exact solution.
            We have two random variables uniformly distributed in [0,1], this means they have a cumulative distribution function (CDF) such as:

            Fe1(x)=Fe2(x)={0,x<0x,0x11,x>1

            We need to calculate the CDF of the random variable e=min(e1,e2) and then by derivation the probability density function (pdf). We know that:

            P(e>x)=1P(ex)=1Fe(x)      (1).

            We can explicitly calculate P(e>x) in this way:

            P(e>x)=P(min(e1,e2)>x)=P(e1>x,e2>x)

            because by definition of minimum each of the two random variables must be greater than x, and this gives us the joint probability of the two variables. The two variables are independent so:

            P(e1>x,e2>x)=P(e1>x)P(e2>x)=(1Fe1(x))(1Fe2(x))=={1,x<0(1x)2,0x10,x>1

            Now we can go back to (1) and find:

            Fe(x)=1P(e>x)={0,x<01(1x)2,0x11,x>1

            and by derivation pdf is:

            fe(x)={2(1x),0x10,elsewhere

            Finally for the expected value of e we have:

            E[e]=+xfe(x)dx=201x(1x)dx=13

            2. btw, a really slick way to do problem 6 is partition your sample space into 2:

              010y1dxdy

              010x1dydx

              If you are familiar with Cauchy's formula for repeated integration, those will jump out at you as each being 12!=12. Thus you know you have summed up the entire sample space because 12+12=1.

              The associated expected value is: 010yxdxdy + 010xydydx
              or by symmetry, we can say = 2010yxdxdy=13

              For fun, we could unpack that value of x and leverage Cauchy's formula instead, getting:

              2010yxdxdy=2010y0x1dwdxdy=2(13!)=13

              Footnote: when I first looked at this problem I saw the 12! so vividly that I thought I could eyeball the answer as being 14... it turns out that this was not an eyeball level problem!

            5. I solved this problem analytically in a similar way, by also integrating the min(y, x) function over both y and x. I did have to search the internet for a way how to do this though, and found the following intuitive derivation:

              The idea is that, to get the expected value, you need to integrate over y as well as over x, so two integrals. The inner integral is over y, and you divide the interval 0 to t in two parts: In the first part y is from 0 to x it's y that is smaller, and the second part y is from x to t, and it's x that is smaller. These are then integrated over y, and then over x. Filling in t=1 provides the answer 1/3.

              I have to admit that my first experiment was to write a Python simulation, so after this I knew the answer had to be 1/3. Then I sought to find the analytical version as well.

                  1. Here is a partial port of my MATLAB code to Python + quadprog, Problem 10 harness omitted for brevity, and not tested on Problems 8-9. Please see the e-Chapter for details.

                    import numpy as np
import quadprog    
SV_CUTOFF = 1e-5


def test_svm():
    '''toy problems from e-Chapter pp. 11, 28'''
    X = np.array([[0, 0], [2, 2], [2, 0], [3, 0]])
    y = np.array([[-1, -1, 1, 1]]).T
    w1 = svm_primal(X, y);
    [alpha, w2] = svm_dual(X, y);        
    print ('primal qp', w1)
    print('dual qp', w2)
    print('dual alpha', alpha)


def svm_primal(X, y):
    '''training examples as rows of X. e-Chapter 8, pp. 9-10'''
    N, d = X.shape

    c = np.ones(N);
    p = np.zeros(d+1);
    Q = np.eye(d+1); 
    Q[0, 0] = 0;   
    A = np.concatenate([y, (y).dot(np.ones((1, d))) * X], axis=1) # outer prod.

    Q[0, 0] += 1e-256 # force it to be positive definite (cantankerous API)
    results = quadprog.solve_qp(G=Q, a=-p, C=A.T, b=c)
    return results[0]


def svm_dual(X, y):
    '''training examples as rows of X. e-Chapter 8, pp. 28, 30'''
    N, d = X.shape
    Xs = (y).dot(np.ones((1, d))) * X 
    Qd = (Xs).dot(Xs.T)
    Ad = np.concatenate([y.T, np.eye(N)])
    pd = np.ones(N);
    cd = np.zeros(N+1);

    Qd += 1e-5 * np.eye(Qd.shape[0]) # force it to be positive definite
    results = quadprog.solve_qp(G=Qd, a=pd, C=Ad.T, b=cd, meq=1)
    alpha = results[0]

    w = np.zeros(d)
    for i in range(N):
        if alpha[i] > SV_CUTOFF * np.max(alpha):
            w += X[i, :].T * y[i] * alpha[i]     

    # (8.24) recover bias from weights + largest support vector
    s = np.argmax(alpha)
    b = y[s] - (X[s, :]).dot(w)

    return [alpha, np.array([b[0], w[0], w[1]])]

                    8. @charlysl

                      I just have one concern, which is that being relatively little used, I fear that I will be less likely to find solutions online if I ran into problems. Almost every time that I had an issue with Matlab I could find a solution in a flash

                      Believe me I know exactly where you are coming from. With Python, just about every issue I've ever run into has been picked over and addressed multiple ways on stackoverflow, etc.

                      With Julia I frequently will find one directly related post or none at all, and you are left on your own to play around with things. All languages have their quirks. Generally my way of learning my way through them is via stackoverflow and experimenting. Sometimes with Julia it feels like I'm just experimenting! (It's not quite this bad, though -- see what I say about Operations Research a bit later in this post.)

                      Yoshi and I went back and forth about this near the bottom of the solutions to week 2. Do a 'find' on yoshi161mi in the below thread to jump to it.

                      https://courses.edx.org/courses/course-v1:CaltechX+CS_1156x+3T2016/discussion/forum/i4x-Caltech-CS156-course-Fall2016_Week2/threads/57fc0ab9bc04a6057f000e09

                      I also wonder about the availability and maturity of libraries, a related issue

                      Roughly speaking: don't worry about this. In practice, this basically means that certain things you do may be deprecated over time (and in future versions you change the command accordingly). For mathematical optimization, Julia is quite good and it has become very popular in Operations Research (OR) departments. This means you have very quality optimization packages and you can find great slides doing optimization with Julia walkthroughs on various OR department websites, via our friend Google.

                      As far as other stuff, like array operations and Linear Algebra, also don't worry about this. Alan Edelman, one of the two main linear algebra dons at MIT, has been a guiding force in Julia since day one -- at least as I understand it.

                      There are some ridiculously powerful libraries already in place. They continue to add more, of course.

                      As far as plotting and visualizations go, Julia pragmatically imports PyPlot -- i.e. it imports the Python plotting module directly. So basically you get Python level plotting automatically with Julia. (FWIW, Python is reputed to have pretty good plotting, but not necessarily as good as R or Matlab.)

                      At the end of the day, I reach for Julia when dealing with sparse data or modelling mathematical optimization problems.

                      Julia is a very cool language. But it's definitely an adventure!

                        SVM <- function(dataset,N,doplot){

   X <- cbind(dat$dataset$X1,dat$dataset$X2)
   Y <- dat$dataset$Y

   Dmat <- MakeD(X,Y,N)
   mod <- 1e-13
   Dmat <- Dmat + mod * diag(N)

   dvec <- t(rep(1,N))
   A <- rbind(Y,diag(N))
   Amat <- t(A)
   bvec <- matrix(rep(0,N+1),nrow=N+1)

   qpr <- solve.QP(Dmat,dvec,Amat,bvec,meq=1)

   alphas <- qpr$solution
       iter <- qpr$iterations[1]

   w <- (Y*alphas)%*%X

   max_alpha_index <- which.max(alphas)
   b <- (Y[max_alpha_index] - w%*%X[max_alpha_index,])

   hw <- c(b,w)

   if (doplot) {
     abline(-hw[1] / hw[3], -hw[2] / hw[3], col = "red")
   }

   return(list("iter" = iter, "hw" = hw, "alf" = alphas))

                        }

                        MakeD <- function(X,Y,n){

                           Dmat <- matrix(0,nrow=N,ncol=N)

   for (j in 1:n){

       for (i in 1:n){

          Dmat[i,j] <- Y[i]*Y[j]*(t(X[i,])%*%X[j,])

     }

   }

   return("DM" = Dmat)

 }
